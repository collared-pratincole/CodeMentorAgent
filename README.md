# CodeMentor AI — 你的 AI 编程私教

## 📌 项目概述

CodeMentor AI 是一款基于 Web 的 AI 编程学习平台，桌面端基于 Electron 打包为独立应用。它集成国内外 15+ 主流 AI 大模型，为用户提供两大核心能力：**项目逆向学习**（丢进任意项目，AI 一步步教你从零构建）和**语言系统学习**（选择一门语言，获得 30 天学习路线，每日引导式课程 + 实战练习 + 考试背记）。目标是让编程学习从"看视频"升级为"跟着 AI 动手做"。

- **目标用户**：编程初学者、转行学习者、想通过阅读开源项目提升能力的开发者
- **核心价值**：AI 驱动的个性化学习路径 + 沉浸式代码编辑环境，让学习与实践零距离
- **数据安全**：所有数据（API Key、学习记录、项目文件）全部保存在你的电脑本地，不上传任何服务器

## ✨ 核心功能

- **仪表盘**：今日学习任务、学习进度环、连续打卡天数、快捷入口一目了然
- **项目逆向学习**：上传任意项目文件夹，AI 自动识别技术栈、生成全景分析报告（项目类型、用途、目标用户、核心功能、价值亮点），并把项目拆成多个「从零构建」步骤，每步包含目标说明 + 原始代码 + 讲解
- **语言学习路径**：选择 14 门主流语言之一，填写背景和每天可用时间，AI 生成 30 天为一月的学习路线；每张卡片代表一天，完成当天后自动解锁下一天
- **每日课程**：知识点讲解（Markdown 渲染）+ 代码示例（Monaco Editor）+ 实战练习 + AI 自动评判打分；每天内容仅生成一次，跨页面跳转状态持久保留
- **代码运行**：内置代码执行能力，10 秒超时 + 100KB 输出限制 + 自动临时文件清理；运行失败时弹窗提示改用「提交」按钮让 AI 检阅打分
- **考试系统**：阶段考试（简答题 + 代码题，独立滑块控制数量）+ 月度考试（完成整月 30 天课程后解锁）；AI 自动批改主观题，分数限制在 0 到满分之间；考后给出四格统计卡 + 双进度条对比 + 规则化改进建议
- **考前速记**：上传资料后 AI 自动生成背记卡片，考前快速复习重点
- **成就系统**：连续打卡、累计经验等里程碑徽章，让坚持可视化
- **AI 对话**：多模型实时切换、代码问答、快捷指令；SSE 流式响应逐字渲染
- **多模型支持**：内置 15+ AI 提供商（OpenAI / Anthropic / DeepSeek / 通义千问 / 文心一言 / 智谱 GLM / Kimi / 零一万物 / 豆包 / SiliconFlow / Ollama / Google Gemini / Groq / 自定义兼容接口）
- **主题与外观**：亮色 / 暗色主题、3 种强调色（琥珀 / 绿色 / 紫色）、编辑器字号可调
- **响应式设计**：桌面端侧边栏导航，移动端自动切换为底部 Tab 栏并隐藏顶栏
- **任务队列**：所有长耗时操作（项目分析、构建步骤生成、考试创建、每日课程生成）进入任务队列，铃铛弹窗实时显示圆形进度与任务详情
- **一键打包**：支持打包为 Windows 独立 EXE 安装程序，无需安装 Node.js 环境即可运行

## 🚀 快速开始（推荐）

### 第一步：下载程序
1. 访问项目的 **Release 页面**；
2. 下载最新版本的 `CodeMentor AI` 安装程序（`.exe`）；

### 第二步：安装并启动
1. 双击运行安装程序，按提示完成安装（可自定义安装路径）；
2. 安装完成后会自动创建桌面快捷方式和开始菜单快捷方式；
3. 首次运行可能被 Windows Defender 拦截，点击「更多信息」→「仍要运行」即可。

### 第三步：初始化设置
1. 启动后会显示加载页，后端服务就绪后自动进入应用；
2. 首次进入会弹出欢迎提示和设置向导；
3. 在「设置」页面配置至少一个 AI 提供商的 API Key（API Key 仅保存在本地后端，前端只显示脱敏预览如 `sk-***...cdef`）；
4. 选择默认模型，即可开始使用。

### 第四步：开始学习
1. **项目逆向**：进入「项目分析」，拖入任意项目文件夹，AI 自动生成全景分析和构建步骤；
2. **语言学习**：进入「学习路径」，选择一门语言，AI 生成 30 天学习路线，每天一节课 + 一个练习；
3. **AI 问答**：进入「AI 对话」，随时和 AI 导师交流编程问题。

## 📦 备用方案：手动部署（开发者模式）

### 部署前提
1. **Node.js 18+** 运行环境（推荐使用 LTS 版本）
2. npm 包管理工具（已随 Node.js 安装）

### 获取项目文件
1. 将项目文件完整下载或克隆至本地目录；
2. 项目根目录包含前端代码（`src/`）、后端服务（`server/`）、Electron 主进程（`electron/`）等，**不要移动或重命名这些文件夹**。

### 安装依赖
1. 打开终端/命令行，进入项目根目录；
2. 安装前端依赖：
   ```bash
   npm install
   ```
3. 安装后端依赖（后端使用独立的 `server/package.json`）：
   ```bash
   cd server && npm install --production && cd ..
   ```
4. 等待安装完成，无报错即表示成功。

### 启动服务

**方式一：下载安装包直接运行（最省事）**
1. 访问项目的 **Release 页面**；
2. 下载最新版本的 `CodeMentor AI` 安装程序（`.exe`）或免安装压缩包；
3. 双击安装程序按提示完成安装，或解压免安装压缩包后双击运行 `CodeMentor AI.exe`；
4. 启动后会自动拉起内置后端服务并打开桌面窗口，无需任何额外配置即可使用（首次进入需在设置中配置 AI API Key）。

**方式二：仅启动 Web 服务（浏览器访问）**
```bash
npm run dev:all
```
该命令会同时启动前端开发服务器（Vite，端口 5173）和后端服务（Express，端口 3001），启动后在浏览器访问：`http://localhost:5173`

**方式三：使用启动脚本（仅后端，配合已构建的前端）**
```bash
# Windows
start.bat
# macOS / Linux
./start.sh
```
该方式仅启动后端服务（端口 3001），需先执行 `npm run build` 构建前端到 `dist/` 目录，后端会自动托管静态文件，访问：`http://localhost:3001`

**方式四：Electron 桌面端开发模式**
```bash
npm run electron:dev
```
该命令会并行启动前端、后端和 Electron 主进程，自动打开桌面窗口。

### 打包为 EXE（可选）
```bash
# 打包为安装程序（NSIS）
npm run dist

# 仅打包为免安装目录（调试用）
npm run dist:dir
```
打包完成后，在 `release/` 文件夹中找到安装程序或免安装目录。

## 🔧 关键技术说明

- **前端框架**：React 18 + TypeScript + Vite 6
- **样式方案**：Tailwind CSS 3 + CSS Variables 主题系统（支持亮/暗色 + 多强调色）
- **代码编辑器**：@monaco-editor/react（VS Code 内核）
- **状态管理**：Zustand（轻量级，多 Store 拆分，含 localStorage 持久化）
- **路由**：react-router-dom 7
- **Markdown 渲染**：react-markdown + remark-gfm + rehype-highlight
- **图表渲染**：mermaid（架构图 / 流程图）
- **图标**：lucide-react
- **动画**：framer-motion
- **本地存储**：IndexedDB（Dexie.js 封装）存储学习进度、对话历史
- **后端服务**：Express 4 + CORS，文件型 JSON 存储（无需数据库）
- **桌面打包**：Electron 43 + electron-builder（NSIS 安装程序）
- **AI 接入**：统一 OpenAI 兼容接口适配，SSE 流式响应（基于缓冲区的行解析，防止数据丢失）；AI 响应解析采用状态机 JSON 修复，保留字符串内部结构
- **端口约定**：后端服务固定使用 3001 端口，前端开发服务器使用 5173 端口

## 📌 重要注意事项

1. **API Key 安全**：API Key 仅保存在本地后端文件中，前端只接收脱敏预览（如 `sk-***...cdef`），不会明文加载、缓存或在其他位置写入；AI 请求统一通过后端代理发出，避免 CORS 问题。
2. **端口占用**：后端服务默认运行在端口 3001，启动前请确保该端口未被占用，否则会报 `EADDRINUSE` 错误。
3. **数据存储**：所有用户数据保存在 `server/data/users/<用户ID>/` 目录下，包括 API Key、项目源码、分析结果、学习进度、考试记录等；Electron 打包后数据保存在系统 userData 目录下，卸载不会清除。
4. **每日课程生成**：每天的课程内容仅生成一次（按自然天），生成状态跨页面跳转持久保留，不会重复消耗 AI 额度。
5. **月度考试解锁条件**：必须完成目标月份全部 30 天每日课程后，月度考试才会开放。
6. **项目删除**：删除项目会弹出确认框二次确认；删除时会先尝试 Node.js `fs.rmSync`，失败后回退到系统命令（Windows 用 `rmdir /S /Q`，Unix 用 `rm -rf`），并做最终存在性检查确保删除成功。
7. **代码运行限制**：代码执行 API 有 10 秒超时、100KB 输出限制，并自动清理临时文件；运行失败时弹窗提示改用「提交」按钮让 AI 检阅打分，而不是死磕本地环境。
8. **推荐浏览器**：Web 模式推荐使用现代浏览器（Chrome / Edge / Firefox / Safari）以获得最佳视觉效果。
9. **EXE 安全提示**：EXE 文件首次运行可能被 Windows Defender 拦截，点击「更多信息」→「仍要运行」即可；本项目完全开源，可自行审查代码或从源码构建。
10. **关闭即停**：关闭 Electron 窗口或终止命令行进程即可停止服务，后端子进程会自动清理。
